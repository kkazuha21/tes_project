# SharedLab 课程作业：提交清单与目录结构（模板）

日期：2026-01-06

> 目的：把“需要提交的材料”和“压缩包内目录结构”一次性写清楚，便于验收与评分。
> 
> 说明：如果你的《测试要求.md》里有更严格的命名/目录/截图要求，以课程要求为准；本文提供一个可直接落地的推荐结构。

---

## 1. 提交物清单（Checklist）

### 1.1 必交文档（Markdown）

- 软件测试计划：local/SharedLab-软件测试计划.md
- 软件测试说明：local/SharedLab-软件测试说明.md
- 软件测试报告：local/SharedLab-软件测试报告.md
- 接口测试分工：local/接口测试分工.md
- 评分表（用例数量统计/材料清单）：local/评分表.md

### 1.2 测试用例与执行记录

- 功能测试用例表（含执行结果/结论/测试人员/时间）：local/功能测试用例表.md
  - 建议补齐：每条用例的“实际结果/结论/截图证据路径/执行人/日期”。

> 如果你们要求“用例汇总表（Excel）”，可以用同样字段导出为表格文件；当前工作区以 Markdown 形式维护。

### 1.3 接口/性能测试脚本与结果

- JMeter 测试计划：tests/jmeter/SharedLab_API_LoadTest.jmx
- JMeter 账号数据：tests/jmeter/data/users.csv
- 一键运行脚本：tests/jmeter/run-jmeter.ps1
- 说明文档：tests/jmeter/README.md

执行后（建议一并提交）：
- JTL 结果：tests/jmeter/results/run_*.jtl
- HTML 报告：tests/jmeter/results/report_*/（文件夹）

> 注意：如果你们还未执行压测，也可以先提交脚本与说明；但“测试报告”里性能指标需要回填实测值。

### 1.4 证据材料（截图/日志）

建议最少包含：
- 功能测试关键路径截图（登录、设备列表、设备详情、我的预约/报修等）
- 关键缺陷/异常用例截图（如：错误密码、未登录 401、越权 403、预约冲突 409 等）
- JMeter 执行截图（命令行运行完成）
- JMeter HTML 报告截图（Dashboard 总览 + Response Times + Statistics）

## 2. 提交压缩包目录结构（推荐）

建议把最终提交物整理为一个压缩包（zip/rar），结构如下：

```
<组名或项目名>-软件测试作业-20260106/
  01-文档/
    SharedLab-软件测试计划.md
    SharedLab-软件测试说明.md
    SharedLab-软件测试报告.md
    接口测试分工.md
    评分表.md

  02-用例与记录/
    功能测试用例表.md

  03-接口与性能测试-JMeter/
    SharedLab_API_LoadTest.jmx
    data/
      users.csv
    run-jmeter.ps1
    README.md
    results/
      run_*.jtl
      report_*/

  04-证据截图/
    功能测试/
    接口测试/
    性能测试/

  05-可选-补充材料/
    （如：缺陷清单、会议纪要、测试数据说明等）
```

> 如果老师要求“必须包含源代码/可执行包”，可以把它放进 `05-可选-补充材料/`，或按课程要求另建 `00-源码/`、`00-可执行/`。

---

## 3. 文件命名规则（建议）

- 文档类：保持统一前缀，例如 `SharedLab-软件测试计划.md`、`SharedLab-软件测试说明.md`、`SharedLab-软件测试报告.md`
- 截图类：建议包含【模块-用例编号-结果-日期-执行人】
  - 示例：`LOGIN-001-成功-20260106-张三.png`
  - 示例：`BOOK-003-冲突409-20260106-李四.png`
- JMeter 结果：保留脚本自动生成的目录即可（`results/run_*.jtl`、`results/report_*`）

---

## 4. 交付自检（提交前 2 分钟检查）

- 三份文档（计划/说明/报告）能互相引用且路径正确
- 评分表里已填：成员姓名/学号/手机号（如需）+ 各类用例数量
- 功能测试用例表里已填：实际结果/结论/执行人/日期 + 至少若干截图证据
- JMeter：
  - `SharedLab_API_LoadTest.jmx` 可被 XML 正常解析
  - 若已执行，`results/` 下存在 `run_*.jtl` 与 `report_*/`
