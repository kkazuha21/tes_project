# 共享实验室平台 - 角色权限系统说明

## 📌 系统概述

系统现已支持三种用户角色，每个角色有独立的界面和功能权限：

### 👨‍🎓 学生 (STUDENT)
- **登录后进入**: `/student/devices`
- **可用功能**:
  - 查看设备列表
  - 预约设备
  - 查看我的预约
  - 提交报修
  - 查看我的报修

### 👨‍🏫 教师 (TEACHER)
- **登录后进入**: `/teacher/devices`
- **可用功能**:
  - 查看和管理设备
  - 查看所有预约记录
  - 处理学生报修申请
  - 自己也可以预约设备
  - 查看自己的预约

### 👨‍💼 管理员 (ADMIN)
- **登录后进入**: `/admin/dashboard`
- **可用功能**:
  - 系统数据概览
  - 用户管理
  - 设备管理
  - 预约管理
  - 报修管理
  - 设备分类管理
  - 系统日志查看

## 🔐 权限控制

### 路由守卫
系统在路由层面实现权限控制：
1. 未登录用户访问任何页面 → 重定向到登录页
2. 已登录用户访问登录页 → 根据角色重定向到对应首页
3. 用户访问非本角色页面 → 重定向到对应角色首页

### 登录流程
1. 用户输入用户名和密码
2. 系统验证并返回用户信息（包含角色）
3. 根据角色自动跳转：
   - ADMIN → `/admin/dashboard`
   - TEACHER → `/teacher/devices`
   - STUDENT → `/student/devices`

## 🔧 技术实现

### 前端路由结构
```javascript
/student/*          - 学生路由（需要 STUDENT 角色）
/teacher/*          - 教师路由（需要 TEACHER 角色）
/admin/*            - 管理员路由（需要 ADMIN 角色）
```

### 状态管理
- 使用 Pinia store 管理用户认证状态
- 保存用户角色信息
- 提供角色判断方法：isAdmin, isTeacher, isStudent

### API 权限
后端 API 应该已经实现了基于角色的权限控制（通过 JWT token 中的角色信息）

## 📝 注意事项

1. **密码加密**: 生产环境中密码必须使用 BCrypt 加密
2. **Token 安全**: JWT token 包含用户角色信息
3. **权限验证**: 前后端都需要验证用户权限
4. **会话管理**: 用户信息保存在 localStorage，刷新页面不会丢失登录状态
