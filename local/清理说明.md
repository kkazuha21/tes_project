# 项目文件清理说明

## 📂 已整理的文件

以下文件夹包含测试时创建的临时文件,确认项目运行正常后可以安全删除:

### 1. `/临时文件_可删除/`
包含项目根目录的旧启动脚本和测试文件:
- check-java.bat
- fix-*.bat / fix-*.sh
- start-*.bat / start.sh
- 各种测试用的 .md 文档

### 2. `/backend/临时脚本_可删除/`
包含后端的测试启动脚本:
- run-dist-mvn25.bat
- run-with-jdk21.bat
- run-with-jdk25.bat
- start-backend-background.bat
- start-backend-jar.bat

## ✅ 保留的重要文件

### 根目录
- `快速启动.bat` - 一键启动脚本 ⭐
- `启动说明.md` - 启动文档 ⭐
- `README.md` - 项目说明
- `PROJECT_OVERVIEW.md` - 项目概览
- `.gitignore` - Git 配置

### /backend/
- `start-backend-jdk21.bat` - 后端启动脚本 ⭐
- `insert-test-data.sql` - 测试数据 ⭐
- `pom.xml` - Maven 配置
- `mvnw.cmd` - Maven Wrapper
- `/src/` - 源代码
- `/target/` - 编译输出

### /frontend/
- `package.json` - 项目配置
- `vite.config.js` - Vite 配置
- `/src/` - 源代码
- `/node_modules/` - 依赖包

### /database/
- `init.sql` - 数据库初始化脚本 ⭐

## 🗑️ 清理建议

1. **测试运行** - 使用 `快速启动.bat` 确保项目正常启动
2. **确认功能** - 登录系统,测试设备管理等功能
3. **安全删除** - 确认无误后,可以删除两个 "临时文件_可删除" 文件夹

## 📌 注意事项

- 不要删除 `node_modules` 和 `target` 目录(虽然可以重新生成,但需要重新下载/编译)
- 保留 `.mvn` 文件夹(Maven Wrapper 配置)
- Git 相关文件(.gitignore)应保留

## 🔄 如需恢复

如果误删重要文件,可以:
1. 从 Git 仓库恢复
2. 重新运行项目初始化脚本
