## 1. 简介

### 1.1 系统概述

共享实验器材平台（SharedLab）为实验室设备的共享与预约提供支撑，核心能力包括：
- 用户认证与角色权限（学生/教师/管理员）
- 设备浏览与设备管理
- 预约调度（含预约时间冲突检测）
- 故障报修与处理

系统形态：前后端分离 Web 应用。

- 明确测试范围、测试类型、测试策略与资源安排

## 2. 测试范围与目标

### 2.1 测试范围

本次计划覆盖：
- UC001 用户权限（登录/注册、权限控制）
- UC002 设备管理（设备列表/详情；管理员设备维护）
- UC003 预约调度（创建预约、冲突检测、取消、查询）
- UC005 故障报修（提交、查询、状态更新）

注：使用记录/系统日志等模块以实际实现为准；如后端接口未提供，则仅做前端可用性与占位说明。

### 2.2 不在范围内（如当前版本未实现/不具备条件）

- 生产环境部署与运维监控验证
- 第三方服务可用性（短信、邮件、对象存储等）
- 安全渗透测试与合规审计（可作为后续扩展）

### 2.3 测试目标

- 功能正确：关键业务流程可完成且异常处理合理
- 鉴权正确：不同角色访问受控页面/API 时行为符合预期
- 稳定可用：基本并发压力下接口可用且错误率可控（若执行性能测试）

## 3. 测试策略

### 3.1 测试类型

1) 功能测试（手工）
- 基于 local/功能测试用例表.md 执行
- 覆盖正向/反向/边界/权限场景

2) 接口测试（JMeter）
- 基于 tests/jmeter/SharedLab_API_LoadTest.jmx
- 覆盖登录、设备浏览、分类/预约/报修、角色接口（管理员/教师）

3) 回归测试
- 缺陷修复后，对相关页面/接口进行复测

### 3.2 测试方法

- 等价类划分、边界值分析、错误推测法
- 重点关注：时间段重叠（预约冲突检测）、权限校验、必填与格式校验

### 3.3 通过准则（建议）

- 功能用例：关键路径用例通过率 100%（登录、设备浏览、预约查询/取消、报修查询）
- 接口测试：
  - HTTP 状态码与响应结构满足断言
  - 401/403 场景可复现且提示合理
- 性能测试（若执行）：在指定并发/时长下错误率不超过阈值；关键接口响应时间满足阈值

> 注：具体性能阈值需以你的《测试要求.md》为准；该文件当前不在工作区内。
