# 共享实验室平台 - 路由空白页面问题排查

## 问题描述
点击菜单后页面显示空白

## 已修复的问题

### 1. 添加了默认子路由重定向
现在访问父路由会自动重定向到默认子路由：
- `/student` → `/student/devices`
- `/teacher` → `/teacher/devices`  
- `/admin` → `/admin/dashboard`

### 2. 路由配置说明

**正确的路由结构：**
```
父路由 (Dashboard 组件)
  └─ 子路由 (具体页面组件)
```

**示例：**
```
/student                    ← StudentDashboard.vue (框架)
  ├─ /student/devices       ← DeviceList.vue (显示在 <router-view /> 中)
  ├─ /student/bookings      ← MyBookings.vue
  └─ /student/tickets       ← MyTickets.vue
```

## 可能的原因

### 1. Vue 组件加载失败
检查浏览器控制台是否有以下错误：
- `404 Not Found` - 组件文件不存在
- `SyntaxError` - 组件语法错误
- `Failed to fetch` - 网络请求失败

### 2. 路由配置问题
- ❌ 只访问父路由 `/student` 
- ✅ 应该访问子路由 `/student/devices`

### 3. <router-view /> 缺失
Dashboard 组件必须包含 `<router-view />` 才能显示子路由内容

### 4. 权限问题
确保登录的角色与访问的路由匹配：
- STUDENT → /student/*
- TEACHER → /teacher/*
- ADMIN → /admin/*

## 常见错误处理

### 错误1: 页面空白但 URL 正确
**原因**: 组件文件不存在或加载失败
**解决**: 检查 Console 中的 404 错误，确认组件文件路径

### 错误2: 跳转后立即重定向到登录页
**原因**: Token 过期或未正确保存
**解决**: 清除 localStorage 重新登录

### 错误3: 侧边栏菜单点击无反应
**原因**: 路由路径配置错误
**解决**: 检查 el-menu-item 的 index 属性是否与路由 path 一致
