# 设备分类功能完善说明

## 完成的功能

### 1. 后端改进

#### 实体类更新 (DeviceCategory.java)
- ✅ 添加 `description` 字段 - 分类描述
- ✅ 添加 `createdAt` 字段 - 创建时间
- ✅ 添加 `updatedAt` 字段 - 更新时间
- ✅ 添加 `@PrePersist` 和 `@PreUpdate` 自动设置时间戳

#### 新增 DTO (CategoryResponse.java)
```java
- id: 分类ID
- name: 分类名称
- description: 分类描述
- deviceCount: 关联设备数量
- createdAt: 创建时间
- updatedAt: 更新时间
```

#### 新增 Service (CategoryService.java)
- `getAllCategories()` - 获取所有分类(含设备数量统计)
- `getCategoryById(Long id)` - 获取单个分类详情
- `createCategory(DeviceCategory category)` - 创建分类
  - 检查名称唯一性
- `updateCategory(Long id, DeviceCategory categoryDetails)` - 更新分类
  - 检查名称唯一性(排除自身)
- `deleteCategory(Long id)` - 删除分类
  - 检查是否有关联设备,有则禁止删除

#### 新增 Controller (CategoryController.java)
```
GET    /api/categories          - 获取所有分类
GET    /api/categories/{id}     - 获取分类详情
POST   /api/categories          - 创建分类 (仅管理员)
PUT    /api/categories/{id}     - 更新分类 (仅管理员)
DELETE /api/categories/{id}     - 删除分类 (仅管理员)
```

#### Repository 更新 (DeviceRepository.java)
- ✅ 添加 `countByDeviceCategoryId(Long categoryId)` - 统计分类下的设备数量

### 2. 数据库改进

#### 表结构更新
```sql
ALTER TABLE device_categories 
ADD COLUMN description VARCHAR(500) AFTER name;
```

## API 测试

使用 Postman 或 curl 测试:

### 获取所有分类
```bash
curl http://localhost:8080/api/categories
```

### 创建分类 (需要管理员token)
```bash
curl -X POST http://localhost:8080/api/categories \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "化学试剂",
    "description": "各类化学试剂和溶液"
  }'
```

### 更新分类
```bash
curl -X PUT http://localhost:8080/api/categories/1 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "显微镜",
    "description": "更新后的描述"
  }'
```

### 删除分类
```bash
curl -X DELETE http://localhost:8080/api/categories/5 \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 注意事项

1. **权限控制**: 只有管理员可以创建、编辑、删除分类
2. **名称唯一性**: 分类名称必须唯一,后端会验证
3. **级联删除保护**: 如果分类下有设备,无法删除
4. **设备数量实时统计**: 每次查询分类时,后端会实时统计设备数量
5. **时间戳自动管理**: 创建和更新时间由 JPA 自动管理
