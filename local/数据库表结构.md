# SharedLab 数据库表结构（数据字典）

> 说明：本文件根据后端 JPA 实体类推导生成（`spring.jpa.hibernate.ddl-auto=update`）。实际数据库中的字段类型/长度可能会因 Hibernate 方言、版本、实际建表历史而略有差异；如需“数据库真实结构”，建议用 `SHOW CREATE TABLE` 逐表核对。

## users（用户表）

| 字段名 | 描述 | 数据类型（MySQL 推导） | 是否主键 |
|---|---|---|---|
| id | 主键ID | BIGINT AUTO_INCREMENT | 是 |
| username | 用户名（唯一） | VARCHAR(50) | 否 |
| email | 邮箱（唯一） | VARCHAR(100) | 否 |
| password_hash | 密码哈希 | VARCHAR(255) | 否 |
| role | 角色（枚举，字符串） | VARCHAR(20) | 否 |
| status | 状态（0 禁用 / 1 启用，默认 1） | TINYINT | 否 |
| created_at | 创建时间 | DATETIME | 否 |

## device_categories（设备分类表）

| 字段名 | 描述 | 数据类型（MySQL 推导） | 是否主键 |
|---|---|---|---|
| id | 主键ID | BIGINT AUTO_INCREMENT | 是 |
| name | 分类名称（唯一） | VARCHAR(100) | 否 |
| description | 分类描述 | VARCHAR(500) | 否 |
| created_at | 创建时间 | DATETIME | 否 |
| updated_at | 更新时间 | DATETIME | 否 |

## devices（设备表）

| 字段名 | 描述 | 数据类型（MySQL 推导） | 是否主键 |
|---|---|---|---|
| id | 主键ID | BIGINT AUTO_INCREMENT | 是 |
| name | 设备名称 | VARCHAR(100) | 否 |
| status | 设备状态（枚举，字符串） | VARCHAR(20) | 否 |
| location | 设备位置 | VARCHAR(200) | 否 |
| description | 设备描述 | TEXT | 否 |
| category_id | 设备分类ID（外键，关联 device_categories.id） | BIGINT | 否 |

## bookings（预约表）

| 字段名 | 描述 | 数据类型（MySQL 推导） | 是否主键 |
|---|---|---|---|
| id | 主键ID | BIGINT AUTO_INCREMENT | 是 |
| start_time | 预约开始时间 | DATETIME | 否 |
| end_time | 预约结束时间 | DATETIME | 否 |
| status | 预约状态（枚举，字符串） | VARCHAR(20) | 否 |
| created_at | 创建时间 | DATETIME | 否 |
| user_id | 用户ID（外键，关联 users.id） | BIGINT | 否 |
| device_id | 设备ID（外键，关联 devices.id） | BIGINT | 否 |

## tickets（工单/报修单表）

| 字段名 | 描述 | 数据类型（MySQL 推导） | 是否主键 |
|---|---|---|---|
| id | 主键ID | BIGINT AUTO_INCREMENT | 是 |
| title | 标题 | VARCHAR(200) | 否 |
| content | 内容 | TEXT | 否 |

| status | 工单状态（枚举，字符串） | VARCHAR(20) | 否 |

| device_id | 设备ID（外键，关联 devices.id） | BIGINT | 否 |
